# 钉钉扫码登录功能实现计划

## 一、功能概述

实现钉钉 OAuth2.0 扫码登录，用户扫码后自动识别身份，无需手动输入姓名。

## 二、技术方案

### 钉钉 OAuth2.0 扫码登录流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   前端页面   │───▶│  钉钉授权页  │───▶│  钉钉回调    │
│  展示二维码  │    │  用户扫码    │    │  携带authCode│
└──────────────┘    └──────────────┘    └──────────────┘
                                               │
                                               ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   返回token  │◀───│  获取用户    │◀───│  后端交换    │
│  前端存储    │    │  详细信息    │    │  access_token│
└──────────────┘    └──────────────┘    └──────────────┘
```

## 四、后端 API 设计

### 1. 获取登录二维码 URL
```
GET /api/auth/qrcode
Response: { "qrcode_url": "...", "state": "..." }
```

### 2. OAuth 回调处理
```
GET /api/auth/callback?authCode=xxx&state=xxx
Response: { "token": "...", "user": { "name": "...", "id": "..." } }
```

### 3. 获取当前用户信息
```
GET /api/auth/me
Header: Authorization: Bearer <token>
Response: { "user_id": "...", "user_name": "...", "avatar": "..." }
```

### 4. 退出登录
```
POST /api/auth/logout
Header: Authorization: Bearer <token>
```

## 六、钉钉开放平台配置

需要在钉钉开放平台配置：

1. **应用类型**：企业内部应用
2. **授权回调域**：`localhost:8913`（开发环境）/ 实际域名（生产环境）
3. **权限申请**：
   - `openid` - 用户唯一标识
   - `userinfo` - 用户基本信息

## 七、环境变量配置

在 `.env` 文件中添加：

```bash
# Web 应用配置
WEB_HOST=0.0.0.0
WEB_PORT=8913
WEB_SECRET_KEY=your-secret-key-here

# 钉钉登录回调地址（根据环境配置）
DINGTALK_REDIRECT_URI=http://localhost:8913/api/auth/callback
```

## 九、安全考虑

1. **Token 管理**：使用 JWT，24小时过期
2. **CSRF 防护**：使用 state 参数防止 CSRF 攻击
3. **Session 存储**：token 存储在 localStorage
4. **权限控制**：未登录用户只能访问登录页面

## 十、预期效果

1. 用户打开管理后台，显示登录页面
2. 用户使用钉钉扫描二维码
3. 扫码成功后自动跳转到主页
4. 顶部显示用户真实姓名
5. 调课审批等操作显示真实操作人
