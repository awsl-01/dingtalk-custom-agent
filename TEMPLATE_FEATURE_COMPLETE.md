## 使用流程

### 流程1：批量导入资产

```
1. 用户发送：资产录入模板
   ↓
2. 机器人发送：Excel 模板文件
   ↓
3. 用户下载模板并填写资产信息
   ↓
4. 用户上传填写好的模板文件
   ↓
5. 用户发送：批量导入资产
   ↓
6. 机器人回复：导入成功，共导入 X 条记录
```

### 流程2：资产盘点

```
1. 用户发送：资产盘点模板
   ↓
2. 机器人发送：Excel 盘点模板
   ↓
3. 用户下载模板，实地盘点并填写
   ↓
4. 用户上传填写好的盘点模板
   ↓
5. 系统自动更新资产状态
```

## 技术实现

### 实现方式

1. **模板生成**：使用 openpyxl 生成 Excel 文件
2. **意图识别**：支持多种关键词匹配
3. **文件发送**：通过钉钉 API 发送文件给用户
4. **数据导入**：支持 Excel 和 CSV 格式

### 问题修复

**问题**：用户发送 "资产录入模板" 时，系统误识别为资源搜索请求，而不是资产管理的模板下载请求。

**原因**：RESOURCE_KEYWORDS 中包含 "模板" 关键词，导致 "资产录入模板" 被识别为资源搜索。

**解决方案**：在 main.py 中添加特殊处理，在检查 is_resource_request 之前先检查资产管理的模板请求：

```python
# 检查是否是资产管理的模板请求（优先级高于资源搜索）
asset_template_keywords = ["资产模板", "资产录入模板", "资产盘点模板", "批量导入模板"]
is_asset_template = any(kw in text for kw in asset_template_keywords)

if is_asset_template:
    # 资产管理的模板请求，不走资源搜索
    logger.info("检测到资产管理模板请求，跳过资源搜索")
    # 继续后面的技能匹配逻辑
else:
    logger.info("检测到资源搜索请求")
    # ... 原有的资源搜索逻辑
```

**验证结果**：
- "资产录入模板" → 识别为资产管理模板请求 (is_asset_template=True)
- "投影仪课件模板" → 识别为资源搜索请求 (is_asset_template=False)
- "教学资源模板" → 识别为资源搜索请求 (is_asset_template=False)
