# 权限管理使用指南

## 概述

本系统支持多组织架构的知识库权限管理，不同职位的用户有不同的搜索权限。

---

## 角色权限说明

| 角色 | 名称 | 可访问内容 | 需要审批 |
|------|------|-----------|---------|
| **admin** | 管理员 | 全部 | 否 |
| **principal** | 校长 | 全部 | 否 |
| **director** | 主任 | 公开、内部、机密 | 否 |
| **teacher** | 教师 | 公开、内部 | 机密内容需要 |
| **student** | 学生 | 公开 | 内部、机密需要 |

---

## 访问级别说明

| 级别 | 名称 | 说明 |
|------|------|------|
| **public** | 公开 | 所有人可访问 |
| **internal** | 内部 | 教师及以上可访问 |
| **confidential** | 机密 | 仅管理员/校长/主任可访问 |

---

## 钉钉命令

### 1. 查看权限配置

```
权限配置
```

显示当前组织的角色权限配置和用户统计。

---

### 2. 查看我的权限

```
我的权限
权限查询
```

显示当前用户的角色和可访问内容。

---

### 3. 添加用户（管理员专用）

**命令格式：**
```
添加用户 姓名:[用户名] ID:[用户ID] 角色:[角色] 部门:[部门] 上级:[上级姓名]
```

**示例：**
```
添加用户 姓名:张三 ID:123456 角色:教师 部门:教务处 上级:李四
添加用户 姓名:李四 ID:789012 角色:主任
```

**角色类型：**
- 管理员 (admin) - 最高权限
- 校长 (principal) - 最高权限
- 主任 (director) - 可访问机密内容
- 教师 (teacher) - 可访问内部内容
- 学生 (student) - 仅可访问公开内容

---

### 4. 设置用户角色（管理员专用）

**命令格式：**
```
设置角色 [用户名] [角色]
```

**示例：**
```
设置角色 张三 教师
设置角色 李四 主任 部门:教务处 上级:张三
```

---

### 5. 设置文件权限（管理员专用）

**命令格式：**
```
设置权限 [文件名/关键词] 为 [权限级别]
```

**示例：**
```
设置权限 薪资方案 为 机密
设置权限 课表 为 内部
设置权限 通知 为 公开
```

**权限级别：**
- 公开 (public) - 所有人可访问
- 内部 (internal) - 教师及以上可访问
- 机密 (confidential) - 仅管理员/校长/主任可访问

---

### 6. 申请访问权限

当普通用户查询受限内容时，系统会提示需要申请。

**申请命令：**
```
申请查询 [查询内容]
```

**示例：**
```
申请查询 教师薪资方案
```

系统会自动：
1. 创建审批请求
2. 通知上级审批
3. 等待上级回复

---

### 7. 审批处理（上级专用）

**查看待审批列表：**
```
审批
```

**批准申请：**
```
同意 approval_[申请ID]
```

**拒绝申请：**
```
拒绝 approval_[申请ID] [拒绝原因]
```

**示例：**
```
同意 approval_1234567890
拒绝 approval_1234567890 该内容仅限管理人员查看
```

## 命令行工具

### 初始化组织权限配置

```bash
python scripts/init_permissions.py init [corp_id] "组织名称"
```

**示例：**
```bash
python scripts/init_permissions.py init ding3f80869f26d4bb44a39a90f97fcb1e09 "武汉简讯科技有限公司"
```

---

### 添加用户

```bash
python scripts/init_permissions.py add-user [corp_id] [user_id] "用户名" --role [角色] --department "部门" --manager [上级ID]
```

**示例：**
```bash
# 添加管理员
python scripts/init_permissions.py add-user ding3f80869f26d4bb44a39a90f97fcb1e09 0256442600461015855 "启拓" --role admin --department "技术部"

# 添加普通用户
python scripts/init_permissions.py add-user ding3f80869f26d4bb44a39a90f97fcb1e09 09283666846729490 "晓融" --role teacher --department "技术部" --manager 0256442600461015855
```

---

### 列出所有用户

```bash
python scripts/init_permissions.py list-users [corp_id]
```

**示例：**
```bash
python scripts/init_permissions.py list-users ding3f80869f26d4bb44a39a90f97fcb1e09
```

## 使用场景

### 场景1：校长查看所有内容

校长角色默认可以查看所有内容，无需申请。

```
用户：查询 教师薪资方案
机器人：📚 知识库检索结果（共 5 条）
        1. 教师薪资方案... [相关度：0.95]
        2. ...
```

---

### 场景2：教师查询受限内容

教师查询机密内容时，系统会提示需要申请。

```
用户：查询 教师薪资方案
机器人：⚠️ 找到 3 条相关内容，但您的权限不足无法查看。
        💡 如需查看受限内容，请回复「申请查询 教师薪资方案」发起权限申请。

用户：申请查询 教师薪资方案
机器人：✅ 申请已提交
        申请编号：approval_1234567890
        审批人：李主任
        请等待上级审批，审批结果将通知您。
```

---

### 场景3：上级审批申请

上级收到审批通知后进行处理。

```
机器人：📋 访问审批请求
        申请人：张老师（教务处）
        申请内容：查询机密级别知识库
        查询关键词：教师薪资方案
        请回复「同意 approval_1234567890」或「拒绝 approval_1234567890」

用户：同意 approval_1234567890
机器人：✅ 审批通过
        申请人：张老师
        查询内容：教师薪资方案
        已通知申请人查询结果。
```

---

### 场景4：管理员设置文件权限

管理员可以设置文件的访问权限级别。

```
用户：设置权限 薪资方案 为 机密
机器人：✅ 权限设置成功
        目标：薪资方案
        权限级别：机密
        更新分块数：5
```

## 注意事项

1. **权限继承**：用户权限由角色决定，角色权限在 `meta.json` 中配置
2. **默认权限**：未设置权限的知识块默认为 `public`（公开）
3. **审批过期**：审批请求默认24小时过期，过期后需重新申请
4. **管理员权限**：管理员和校长拥有最高权限，无需审批即可访问所有内容
5. **数据隔离**：不同组织架构的知识库完全隔离，互不影响

## 常见问题

### Q1: 如何查看当前有哪些用户？

A: 使用命令 `权限配置` 或命令行 `python scripts/init_permissions.py list-users [corp_id]`

### Q2: 如何修改用户的角色？

A: 管理员可以使用命令 `设置角色 [用户名] [新角色]`

### Q3: 如何给文件设置权限？

A: 管理员可以使用命令 `设置权限 [文件名/关键词] 为 [权限级别]`

### Q4: 普通用户如何查看受限内容？

A: 使用命令 `申请查询 [内容]` 发起权限申请，等待上级审批

### Q5: 审批请求过期了怎么办？

A: 重新发起申请即可，系统会创建新的审批请求
